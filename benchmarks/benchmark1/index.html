Yiikes ++
Game Design Document
by Henry LI, ChuangWei Ma, Erik Zhou
INTRODUCTION
This document describes a game called "Yiikes ++", which is designed to be a simple, 2D
top-down puzzle game. The game will employ tiled backgrounds, collision detection, arcade
physics, AI, and other 2D game techniques.
TECHNOLOGY
Yiikes ++ will be developed for the WebPlatform using the Phaser 3 game engine, which is a
game engine developed by Photon Storm for developing complex 2D games. All rendering,
music, and sound effects will be handled through Phaser 3. Artwork will be simple, and we’ll use
Piskel for sprites and tiles. We will use Tiled to make levels.
BACK STORY
Noob is a janitor working in the world’s largest underground crime syndicate. He’s been working
there for many many decades. Day after day, he clock’s in and out of work, goes home, eats,
sleeps and wakes up for the next day at work. His life is mundane, he has spent more time
cleaning out toilets than anything else. Except on his 50
th
year in this world, he gets called in by
his manager. Noob has always gotten along with everyone, what does upper management need
from him? It turns out after 20 years of being the janitor, the crime syndicate isn’t as big
anymore. All of their drug dens, prostitution rings, and human trafficking systems all got taken
down by the government. They can’t afford a janitor anymore and offers Noob a chance to make
enough money to retire. Noob was to steal very valuable items from banks and government
facilities. Without his youth, Noob finds this to be ridiculous, but technology has advanced a lot
since Noob’s prime. Noob was given a teleportation gun to help him on his thefts. Apparently
the teleportation gun only works for short distances and is taxing on the body, therefore it hasn’t
been very popular for practical use. He can’t even teleport through solid objects. Noob has no
choice but to take this job, or get killed. You know because he works for criminals.
OBJECTIVE
Noob’s task is to navigate through level from point A to point B, where point B is the endzone.
Noob will be given three shots for teleport in a level and a spell “ghost” to boost him up.
Hopefully, these abilities will allow him to avoid all the crazy obstacles and to land in the
endzone safely. As soon as Noob lands in the endzone, he will move on to his next challenge.
GAMEPLAY
The game will work like a top down puzzle game, with Noob able to move left, right, up, and
down in open space.
Noob will have to be extra careful because there are two different kinds of surfaces, and four
different kinds of obstacles. Each obstacle will damage Noob and reset him back to point A of
the current level. Surface/Collidable will work as follows:

CONTROLS
This game will be played using a keyboard. Once started, use the following:
● W / UP - Move Up
● A / LEFT - Move Left
● S / DOWN - Move Down
● D / RIGHT  - Move Right
● SPACE - Shoot a projectile (Press again to teleport to the projectile)
● G - Increase speed for a short duration
● P / ESC - Pause the game presents a pop-up window to the player asking them to
continue when they are ready. If a game is not in progress, nothing will happen
GRAPHICAL USER INTERFACE
As far as the GUI is concerned, we consider:
● Splash Screen - The splash screen GUI simply presents our game logo and the loading
bar. Upon fully loading, the loading bar will change to a “Click Anywhere to Play” text.
Upon clicking anywhere, there will be a brief scripted sequence played explaining a little
about Noob and the backstory for Yiikes. There will be a skip option after a brief period.
When the backstory is over, it will lead to the Main screen.
● Main Screen - We'll have a game menu that allows the player to select from the
following options:
○ Continue - Will go back to the last level in progress. This button will be disabled
if there isn’t a level in progress.
○ Level Select - If pressed, the game will display the level select screen.
○ Control - If pressed, the game will display the control screen.
○ Help - If pressed, the game will display the help screen.
○ Exit - If pressed, the game program will exit.
● Level Select Screen - The level select screen displays all the available levels. The
levels will be locked if the previous level is not beaten. Level 1 is always unlocked.
● Control Screen - The control screen displays all the controls in the game.
● Help Screen - The help screen displays three sections:
○ Instructions on how to play the game
○ Goals of the game
○ Backstory of the game
● Paused Menu - The menu that shows up when the game is paused.
○ Resume - Ends the paused state and resumes the game
○ Display Controls - If pressed, the game will display an info screen that includes
a description of all game controls.
○ Main menu - Returns to the main menu.
○ Exit - If pressed, the game program will exit.
● In-Game GUI - Not much will be displayed in game. Only the number of bullets Noob
has left and the time. The time is used for best times.
ARTWORK
All artwork in the game will be original. The following needs to be created:
● Noob - Our MC, a simple geometric shape. There will be animations for:
○ Walking Right
○ Walking Left
○ Walking Up
○ Walking Down
○ Shooting
○ Teleporting
○ Dying
○ Speeding up
● Moving Obstacle Sprites - These sprites will move in a pattern. They can kill Noob
upon collision. There will be animations for:
○ Walking Right
○ Walking Left
○ Walking Up
○ Walking Down
● Spikes - A stationary obstacle sprite which kills Noob when Noob passes over it
● Water - A stationary obstacle sprite which drags Noob’s velocity as Noob passes over it
● Normal Wall - An obstacle sprite which only the projectiles can pass through
● Laser Wall - An obstacle sprite which kills Noob upon collision. Any other object can
pass through it
● Spawning Port - A stationary sprite that spawns Noob upon death, new game, or new
level
● Endzone - A stationary sprite that indicates the endpoint of the level. Move to the next
level when Noob is fully inside it.
● Background Tiles - A non-collidable layer in the background. Will be black and white
tiles arranged into a chessboard-like pattern.
SOUND EFFECTS
All sound effects will be original. Sounds must be made to coincide with each of the following
events:
● Shooting a projectile
● Teleporting to the projectile
● MC colliding with laser wall
● MC colliding with moving obstacles
● MC in water
● Death
● Clearing a level
● Starting a new level
● Using ghost (G)
